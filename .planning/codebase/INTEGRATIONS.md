## APIs & External Services

**Degenerus Backend API:**
- API endpoint: `https://api.degener.us`
- Purpose: Discord authentication, affiliate code generation/validation, player state management
- Client: Browser Fetch API (vanilla JS)
- Auth: Credentials-based (session cookies)

## Data Storage

**SQLite Databases:**
- `events.db` - Blockchain event indexing and caching
  - Tables: events, sync_state, contracts, event_indexed_args
  - Location: Configured via `event-indexer/config.example.json` (`db_path`)
  - Rebuilt by: `event-indexer/event_indexer.py`

- `analysis.db` - Contract state analysis and game mechanics tracking
  - Tables: level_tickets, jackpot_results, jackpot_ticket_wins, trait_entries, coinflip_deposits, lootbox_opens, etc.
  - Location: Configured via `event-indexer/config.example.json`
  - Used by: `db/export-jackpot-data.js`

**Static JSON:**
- `beta/jackpot-data.json` - Exported player data (jackpot wins, trait ownership, coinflip results)
  - Generated by: `db/export-jackpot-data.js`
  - Source: analysis.db + events.db via sqlite3 CLI

## Blockchain RPC Endpoints

**Required for Event Indexing (`event-indexer/event_indexer.py`):**
- `rpc_ws` - WebSocket endpoint for live event streaming
  - Example: `ws://localhost:8545` (local node) or external provider
  - Required for: Real-time event subscription via websockets
  - Reconnection: Configurable delay (default 5 seconds)

- `rpc_http` - HTTP endpoint for historical queries and backfills
  - Example: `http://localhost:8545` (local node) or external provider
  - Required for: Block number lookups, ABI loading, historical event backfilling
  - Batch size: Configurable (default 1000 blocks)

**Health checks:**
- Interval: 30 seconds (configurable)
- Threshold: 3 consecutive failures triggers reconnection

## Authentication & Identity

**Discord OAuth:**
- Provider: Discord
- Flow: `https://api.degener.us/auth/discord` (initiates login)
- Callback: Handled by backend API
- Session: Stored as HTTP-only cookie
- Endpoints:
  - `GET /auth/discord/me` - Fetch current user (session required)
  - `POST /auth/discord/logout` - Clear session

**Wallet Address:**
- Connected via MetaMask/EIP-6963 provider injection
- Stored in browser memory (not persisted to server)
- Used for: Contract interactions, transaction signing
