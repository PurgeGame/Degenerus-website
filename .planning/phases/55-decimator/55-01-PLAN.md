Copy directly from 55-RESEARCH.md Section 8:

- **Endpoint:** `GET /player/:address/decimator?level=N&day=M`
- **Path params:** `:address` (42-char hex, lowercased via addressParamSchema)
- **Query params:** `level` (required, integer >= 1), `day` (optional, integer >= 1)
- **Response shape:**
  ```json
  {
    "level": 20,
    "player": "0x71be63f3384f5fb98995898a86b02fb2426c5788",
    "bucket": 7,
    "subbucket": 3,
    "effectiveAmount": "5832100000000000000000",
    "weightedAmount": "833157142857142857142",
    "winningSubbucket": 3,
    "payoutAmount": "41235700000000000000",
    "roundStatus": "closed"
  }
  ```
- **Field notes:**
  - `level`: echo of `?level=N`
  - `player`: echo of `:address`, lowercased
  - `bucket`: integer 2-12 (centennial) or 5-12 (normal); null if player did not burn at this level [document range in Pitfall 1 section of spec; CONTEXT D-03 error]
  - `subbucket`: integer 0 to bucket-1; null if player did not burn
  - `effectiveAmount`: wei-encoded BURNIE decimal string; base + boost + boon accumulated at this level; "0" if no burn
  - `weightedAmount`: wei-encoded BURNIE decimal string; `effectiveAmount / bucket` via BigInt divide; "0" if no burn
  - `winningSubbucket`: integer 0 to bucket-1; null if round not resolved OR not_eligible
  - `payoutAmount`: wei-encoded ETH decimal string; "0" if player did not burn OR subbucket != winningSubbucket
  - `roundStatus`: one of `"open"` | `"closed"` | `"not_eligible"` (THREE states -- no "skipped" because coinflip-loss gate applies to BAF only)

- **Edge cases table:**
  | Condition | bucket | subbucket | effectiveAmount | weightedAmount | winningSubbucket | payoutAmount | roundStatus |
  |-----------|--------|-----------|-----------------|----------------|------------------|--------------|-------------|
  | Player burned, round open | 7 | 3 | "5832e21" | "833e21" | null | "0" | "open" |
  | Player burned, round closed, won | 7 | 3 | "5832e21" | "833e21" | 3 | "4.12e19" | "closed" |
  | Player burned, round closed, lost | 7 | 4 | "5832e21" | "833e21" | 3 | "0" | "closed" |
  | Player did NOT burn (any state) | null | null | "0" | "0" | null or N | "0" | "open" / "closed" |
  | Level has NO decimator round | null | null | "0" | "0" | null | "0" | "not_eligible" |

- **roundStatus derivation:** Three states:
  1. `"not_eligible"`: `NOT EXISTS decimator_rounds WHERE level = ?` AND `NOT EXISTS decimator_burns WHERE level = ?`
  2. `"closed"`: `EXISTS decimator_rounds WHERE level = ? AND resolved = true`
  3. `"open"`: rounds row exists but not resolved, OR burns exist without rounds row yet (indexer lag)

- **Contract-Call Map:** 9-row table mapping each response field to its primary source table (echo | decimator_burns.bucket | decimator_burns.subBucket | decimator_burns.effectiveAmount | derived `effectiveAmount / bucket` | decimator_winning_subbuckets.winningSubBucket | decimator_claims.ethAmount OR derived pool*playerBurn/totalBurn | See roundStatus derivation above).

- **Proposed Backend Implementation:** Full Fastify handler with:
  - Imports (decimatorBurns, decimatorBucketTotals, decimatorWinningSubbuckets, decimatorRounds, decimatorClaims from db/schema/decimator.js; decimatorQuerySchema + decimatorPlayerResponseSchema from schemas/player.js; addressParamSchema + errorResponseSchema)
  - Day-resolution via daily_rng lookup (mirrors INTEG-01 + INTEG-05 pattern)
  - Player burn row query (decimator_burns filtered by (player, level))
  - Weighted amount BigInt divide
  - Round status 3-branch derivation (decimator_rounds + decimator_burns two-table query)
  - Winning subbucket lookup (only when closed + bucket known; decimator_winning_subbuckets filtered by (level, bucket))
  - Payout amount lookup with two-path resolution: prefer authoritative decimator_claims.ethAmount, fallback to computed `pool * playerBurn / totalBurn` using decimator_rounds.poolEth + decimator_bucket_totals.totalBurn
  - Returns {level, player, bucket, subbucket, effectiveAmount, weightedAmount, winningSubbucket, payoutAmount, roundStatus}

- **Zod schemas:**
  ```typescript
  export const decimatorQuerySchema = z.object({
    level: z.coerce.number().int().min(1),
    day: z.coerce.number().int().min(1).optional(),
  });

  export const decimatorPlayerResponseSchema = z.object({
    level: z.number().int(),
    player: z.string(),
    bucket: z.number().int().nullable(),
    subbucket: z.number().int().nullable(),
    effectiveAmount: z.string(),
    weightedAmount: z.string(),
    winningSubbucket: z.number().int().nullable(),
    payoutAmount: z.string(),
    roundStatus: z.enum(['open', 'closed', 'not_eligible']),
  });
  ```

- **Error modes table:**
  | Status | Condition | Response |
  | 200 | Player burned at level | Full payload with non-null bucket + subbucket |
  | 200 | Player did NOT burn | Full payload, bucket=null, subbucket=null, amounts='0' |
  | 200 | Level has no round AND no burns | Full payload, all nulls/zeros; roundStatus='not_eligible' |
  | 400 | level missing or invalid | Zod validation failure |
  | 404 | ?day=M supplied but day does not exist in daily_rng | day_not_found |
